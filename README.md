# Wikipedia UI Language Switch Automation

Проєкт автоматизації тестового завдання з використанням **Playwright + TypeScript**.

## Мета

Автоматизувати перевірку сценарію зміни мови інтерфейсу в вебдодатку **Wikipedia** для авторизованого користувача.

---

## Що реалізовано

У проєкті реалізовано:

- автотест позитивного сценарію зміни мови інтерфейсу авторизованим користувачем
- негативний тест на помилку логіну з неправильним паролем
- негативний тест на спробу доступу до Preferences без авторизації
- використання `process.env` змінних для зберігання чутливих даних
- можливість запуску в Docker
- генерація HTML-звіту Playwright
- копіювання звіту з контейнера в локальну папку хост-машини

---

## Автоматизований тест-кейс

### Назва

Зміна мови інтерфейсу авторизованим користувачем у Wikipedia

### Передумови

- існує валідний обліковий запис Wikipedia
- користувач не заблокований
- користувач має доступ до налаштувань профілю
- у змінних оточення задані `WIKI_USERNAME` та `WIKI_PASSWORD`

### Кроки

1. Відкрити сторінку логіну Wikipedia
2. Авторизуватися валідним користувачем
3. Перейти у розділ **Preferences**
4. Відкрити вкладку **User profile**
5. У секції **Internationalisation** вибрати іншу мову інтерфейсу
6. Зберегти зміни

### Очікуваний результат

- мова інтерфейсу змінюється на вибрану користувачем
- сторінка відображається новою мовою
- значення мови в налаштуваннях відповідає вибраному значенню
- заголовок сторінки відображається відповідною мовою

## Покриті сценарії

### Positive

**Authorized user changes language from Eng to Rus**

- логін валідним користувачем
- перехід у Preferences
- зміна мови інтерфейсу з English на Russian
- перевірка, що в налаштуваннях встановлено `ru`
- перевірка, що локалізований заголовок сторінки відображається російською мовою

### Negative

**Login fails with invalid password**

- спроба логіну з валідним username і невалідним password
- перевірка, що користувач залишається на сторінці логіну
- перевірка, що показано повідомлення про помилку

### Negative

**Unauthorized user opens Preferences**

- неавторизований користувач переходить напряму на `/wiki/Special:Preferences`
- перевірка, що замість сторінки налаштувань відкривається сторінка логіну
- перевірка попередження про необхідність авторизації

---

## Зберігання чутливих даних

- Чутлива інформація не зберігається в коді репозиторію.
- Для авторизації використовуються змінні оточення:
  **WIKI_USERNAME**
  **WIKI_PASSWORD**
- Значення зберігаються у .env

### Приклад .env

- WIKI_USERNAME=your_wikipedia_username
- WIKI_PASSWORD=your_wikipedia_password
- WIKI_BASE_URL=https://en.wikipedia.org

## Як користуватися проєктом

### 1. Клонувати репозиторій

```bash
git clone <repo_url>
cd WikiTestProject
```

### 2. Встановити залежності

```bash
npm install
```

### 3. Встановити браузери Playwright

```bash
npx playwright install --with-deps chromium
```

### 4. Створити .env

Створити файл .env у корені проєкту та додати туди:

- **WIKI_USERNAME=your_wikipedia_username**
- **WIKI_PASSWORD=your_wikipedia_password**
- **WIKI_BASE_URL=https://en.wikipedia.org**

### 5. Запуск усіх тестів

```bash
npm test
```

### 6. Запуск у headed mode

```bash
npm run test:headed
```

### 7. Відкрити HTML-звіт

```bash
npm run test:report
```

### 8. Запуск у Docker

Проєкт налаштований для запуску тестів у Docker-контейнері.

```bash
docker compose up --build
```

або

```bash
docker-compose up --build
```

### 9. Звіт у Docker

- Після виконання тесту в контейнері HTML-звіт не залишається тільки всередині контейнера, а копіюється в локальну папку хост-комп’ютера.
- Локально звіт буде доступний у папці:
  **playwright-report/**
- Також технічні артефакти можуть зберігатися у папці:
  **test-results/**
